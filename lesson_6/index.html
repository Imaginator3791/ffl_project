Внутренний отступ. Padding

Padding - это внутренний отстут от границ объекта до границ содержимого внутри этого объекта.

Задается ТОЛЬКО ПОЛОЖИТЕЛЬНЫМ ЧИСЛОМ.

Единицы измерения - px / % / em / rem.

При указании padding: в px; Например padding: 100px; В зависимости от размера объекта динамически изменяется размер содержимого, отступ же при этом остается фиксированным - неизменным.

При указании padding: в %; Например padding: 20%; Значение padding в % высчитывается от ширины родителя. В зависимости от размера ширины родителя динамически изменяется размер И содержимого И отступа padding.

Существует несколько вариантов подключения padding

Перечисление в НЕСКОЛЬКО строк свойств внутреннего отступа с указанием верх 20px - право 30px - низ 40px - лево 100px.
Сокращенная запись в ОДНО значение = padding: 50px; - отступ со всех сторон по 50px.
Сокращенная запись в ДВА значения = padding: 30px 20px;- отступ сверху и снизу - 30x / слева и справа по 20px.
Сокращенная запись в ТРИ значения = padding: 60px 150px 40px; - отступ сверху 60рх / слева и справа по 150рх / снизу 40рх.
Сокращенная запись в ЧЕТЫРЕ значения, указанная в одной строке. Очередность указания как на часах 12 часов - верхний / 3 часа - правый / 6 часов - нижний / 9 часов - левый. В примере верхний 30рх / правый 50рх / нижний 40рх / левый 60рх padding: 30px 50px 40px 60px.

Картинка имеет фиксированну ширину 800px. Поскольку ширина фиксированная, при уменьшении размера родителя картинка наползает на правый паддинг.

Важно!!! Левый padding для картинки работает всегда!!! Если размер картинки превышает [размер содержимого - то отчего считается padding] и [применяемых внтуренних отступов], то картинка вначале наползает сверху на правый padding.

Если родитель и вьюпорт и далее уменьшается, то картинка вылезет за его пределы и появится полоса прокрутки снизу.

ВАЖНО!! padding для картинки не задается!

Внешний отступ. Margin

margin: ; - это внешний отступ, расстояние от границы объекта до другого объекта.

margin: ; - работает ТОЛЬКО для БЛОЧНЫХ объектов.

Значение margin задаются единицами измерения - px / % / em / rem;

Значение margin может быть как положительным, так и отрицательным!

При использовании отрицательного margin текущий объект поднимается вверх и ВСЕ объекты следующие за ним ТАКЖЕ поднимаются вверх. Важно помнить это свойство отрицательного margin.

Используя внешний отступ / margin: 0 auto; / можно отцентрировать объект по горизонтали. При выполнении следующих условий:

  • Объект должен быть блочным
  • Объект должен быть Уже по ширине чем родитель
  • Объекту задано выравнивание = margin: 0 auto;

Используя внешний отступ / margin: 0 0 0 auto; / можно ПРИБИТЬ блок к правой стороне.

СХЛОПЫВАНИЕ margin.

Если margin задан и снизу и сверху, то он накладывается друг на друга или "схлопывается". При этом если верхний margin 100px, а нижний margin 140px, то схлопывание происходит по бОльшему значению и внешний отступ между объектами составит 140px.

Если верхний margin 50px и нижний margin 50px, то в результате схлопывания внешний отступ между объектами составит 50px.

Для блочно-строчных объектов margin НЕ СХЛОПЫВАЕТСЯ, а суммируется. Но для упрощения расчетов лучше всегда выставлять только нижний margin.

Существует несколько вариантов подключения margin

Перечисление в НЕСКОЛЬКО строк свойств внешнего отступа margin-top - 30px / margin-right - 40px / margin-bottom - 35px / margin-left - 45px.
Сокращенная запись в ОДНО значение = margin: 80px; - отступ со всех сторон по 80px.
Сокращенная запись в ДВА значения = margin: 30px 20px;- отступ сверху и снизу - 30x / слева и справа по 20px.
Сокращенная запись в ТРИ значения = margin: 60px 150px 40px; - отступ сверху 60рх / слева и справа по 150рх / снизу 40рх.
Сокращенная запись в ЧЕТЫРЕ значения, указанная В ОДНОЙ строке. Очередность указания как на часах 12 часов - верхний / 3 часа - правый / 6 часов - нижний / 9 часов - левый. В примере верхний 30рх / правый 50рх / нижний 40рх / левый 60рх margin: 30px 50px 40px 60px.

Вычисление общего размера. box-sizing

По умолчанию размером элемента считается размер контентной области. Если указываешь width, height и padding, то браузер посчитает размер элемента как width + padding * 2 и height + padding * 2.

При помощи свойства box-sizing можно изменить то, как браузер будет рассчитывать размеры элемента.

В качестве значения для свойства box-sizing невозможно использовать что-то кроме ключевых слов. Значение может быть только одно.

Доступные значения:

  • - content-box — значение по умолчанию. Финальная ширина = ширина + правый внутренний отступ + левый внутренний отступ + правая рамка + левая рамка
  • - border-box — значение width / height являются финальными размерами элемента. Финальная ширина = ширина

Из-за стандартного механизма расчёта размера элемента многие начинающие разработчики получают не те размеры элемента, которые ожидали.

При этом есть вариант решения: В самом начале работы «сбросить» стандартное значение, заменив его на предсказуемое `box-sizing`.

Для «сброса» можно использовать универсальный селектор `*`. Напиши в самом начале файла со стилями:
css
* {
box-sizing: border-box;
}

Теперь все размеры элементов будут равны тем значениям, что заданы в свойствах `width` / `height`.

Важно! Данная технология работает ТОЛЬКО с внутренними отступами, за счет уменьшения размера контента. При применении внешних отступов margin, размер конечно будет размер блока + внешний отступ

OVERFLOW - Управление переизбытком контента

Свойство overflow позволяет буквально определить, что делать с содержимым блочного элемента (то есть для элемента, у которого display определяется как block, inline-block, flex или grid) , если оно не влезает в размеры — отобразить или обрезать (с полосами прокрутки или без).

Как пишется:
selector {
overflow: visible;
overflow: hidden;
overflow: clip;
overflow: scroll;
overflow: auto;
}

visible — содержимое отображается снаружи родительского блока, если его размеры больше размеров «родительского» блока (значение по умолчанию);

hidden — содержимое, выходящее за пределы «родительского» блока, обрезается по его границам без прокрутки;

scroll — контент обрезается по границам «родительского» блока, но внутри этой области содержимое доступно с помощью прокрутки;

auto — если содержимое переполняет блок, контент будет доступен для прокрутки.

ВАЖНО!!! Полоса скролла считается внутри объекта!!! не снаружи!!!.

ВАЖНО!!! overflow используется в сочетании с min-height (когда высота объекта ограничена) то есть нам нужно исходя из задачи поместить большой по размеру контент в набольшой по высоте блок .

ВАЖНО!!! overflow используется для работы с картинкой большого размера, если нужно ее спозиционировать на странице, а лишнее по дизайну обрезать .

DISPLAY - Управление типом объекта

По умолчанию все элементы в HTML бывают блочными и строчными. Но в вёрстке часто случаются ситуации, что нам нужно изменить свойства элемента и сделать его не строчным, а блочным. И наоборот.

Тут на помощь приходит свойство `display`. Слово **display** переводится с английского просто и ожидаемо — показывать.

Помимо значений `block` (блочное отображение) и `inline` (строчное отображение) существует смешанное значение `inline-block`.

Значения свойства, которые встречаются в работе чаще всего:

- `none` — полностью скрывает элемент со страницы, не удаляя его при этом из HTML-разметки.
Важно!! Если нужно скрыть объект на странице, лучше использовать аттрибут HIDDEN в свойствах объекта.

- `block` — элемент ведёт себя как блочный.

- `inline-block` — элемент ведёт себя снаружи как строчный, а внутри как блочный.

Если пишешь display: inline-block с целью выстроить несколько блоков в строку, то обязательно указывай для родительского элемента font-size: 0. В противном случае между инлайн-блочными элементами будут небольшие отступы.

Ссылки изначально являются строчными элементами. Для изменения размера ссылки и для возможности задать ей фон её, ссылку, нужно сделать блочной (display: block) или строчно-блочной (display: inline-block).
